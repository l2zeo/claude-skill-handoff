# claude-skill-handoff

[English](./README.md) · **한국어**

긴 세션을 정리하고 새 세션에서 작업을 이어갈 때 사용하는 **검증된 의미론적 인수인계
산출물(reviewed semantic handoff artifact)** 을 만들어주는 Claude 스킬입니다.

> Handoff는 **세션을 1:1로 교체**하기 위한 도구입니다. 병렬 작업, 문서화,
> 대화 기록 보존을 위한 도구가 아닙니다. 긴 컨텍스트의 비용 문제를 해결하기
> 위해 존재합니다 — 무거워진 세션을 끝내고, 깨끗한 세션을 시작하며, 새 세션이
> **꼭 필요한 것만** 물려받게 합니다.

---

## 이 스킬이 필요한 이유

세션은 컨텍스트를 누적합니다. 어느 시점을 지나면 그게 부담이 됩니다:

- 매 턴 토큰 비용이 증가
- 응답 속도 저하
- 할루시네이션(hallucination) 위험 상승
- `/compact`가 실패하거나, 핵심 추론(reasoning)을 잘라먹는 경우

본능적으로는 "그냥 계속 가자"가 되지만, 더 나은 선택은 **세션을 종료하고 새로 시작**
하는 것입니다 — 단, 새 세션이 이전 세션이 멈춘 지점에서 자연스럽게 이어받을 수
있을 때만요. 이 스킬이 만들어내는 결과물이 바로 그 다리 역할을 합니다: 압축되고,
사람이 검증한 상태 전달용 산출물. 새 세션은 이 파일을 첫 입력으로 읽고 시작합니다.

Handoff는 **재도출 비용이 큰 것**(결정, 추론 근거, 실패한 접근, 미해결 이슈)을
보존하고, **그렇지 않은 것**(시간순 서술, 폐기된 탐색, 잡담)을 버립니다.

---

## 스킬이 하는 일

스킬이 발동되면 다음 5단계 파이프라인을 실행합니다:

1. **Scope (범위)** — 무엇이 전달되어야 하고, 무엇이 전달되지 말아야 하는지 확인.
2. **Analyze (분석)** — 결정, 추론, 파일, 실패, 미해결 이슈를 추출.
3. **Draft (초안)** — draft 템플릿을 채워 채팅에 표시.
4. **Verify (검증)** — 사용자에게 승인 · 수정 · 민감 정보 마스킹을 요청. **필수**.
5. **Finalize (확정)** — `handoff-YYYYMMDD-HHMM.md` 파일을 OS 임시 디렉토리에
   저장하고, 절대 경로와 새 세션을 위한 한 문단짜리 킥오프 프롬프트를 출력.

스킬은 길이 제한(목표 150–400줄, 상한 ~600줄)을 강제하며, 저장소(repo)에 저장하는
것을 거부하고, 시크릿 · API 키 · PII를 기본적으로 마스킹합니다.

## 설치

### Claude Code

스킬은 `~/.claude/skills/` (사용자 단위) 또는 `.claude/skills/` (프로젝트 단위)에
설치합니다.

```bash
# 적당한 위치에 레포 클론
git clone https://github.com/l2zeo/claude-skill-handoff.git
cd claude-skill-handoff

# 사용자 단위로 설치
mkdir -p ~/.claude/skills
cp -r handoff ~/.claude/skills/

# 확인
ls ~/.claude/skills/handoff
# SKILL.md  assets/
```

프로젝트 단위 설치:

```bash
mkdir -p .claude/skills
cp -r /path/to/claude-skill-handoff/handoff .claude/skills/
```

Claude Code 세션을 재시작하면(또는 새 세션을 시작하면) 스킬이 활성화됩니다.
별도 등록이나 활성화 절차는 필요 없습니다 — Claude가 해당 폴더의 `SKILL.md`
파일을 자동으로 인식합니다.

### Claude.ai (웹/데스크톱)

설정(Settings) → Capabilities → Skills 에서 `handoff/` 폴더를 스킬로
업로드하거나, zip으로 묶어 `.skill` 파일로 업로드하세요.

## 사용법

### 발동(invocation)

다음 중 어떤 방식이든 동작합니다:

```
/handoff
```

```
이 세션 너무 길어졌어. handoff 만들어줘.
```

```
이 채팅 비용이 점점 비싸지네. 새 세션으로 넘기자.
```

발동되면 스킬이 범위(무엇을 옮길지, 무엇을 빼야 할지, 새 세션의 미션이 무엇인지)를
먼저 물어보고, 초안(draft)을 작성한 뒤, 검증을 요청합니다.

### 전체 흐름

```
You:    /handoff WDS Text 마이그레이션만 다음 세션에서 마무리할 거야.
        Tamagui 쪽 실험 흔적은 빼줘.

Claude: [범위 확인]
        [세션을 조용히 분석]
        [채팅에 draft handoff를 표시하고 4개의 검증 질문 제시]

You:    [승인 / 수정 요청 / 취소]
        ↓
Claude: [승인 시] /tmp/handoff-20260522-1430.md 에 저장
        절대 경로 + 새 세션용 킥오프 프롬프트 출력
```

### 새 세션에서 이어가기

```bash
# 현재 세션 종료
/exit

# 새로 시작
$ claude

# 저장된 handoff 파일을 가리키기
You: /tmp/handoff-20260522-1430.md 읽고 이어서 작업해줘.
```

새 세션은 handoff를 읽고 곧바로 Suggested Next Action #1부터 시작합니다.

## 사용하지 말아야 할 경우

다음과 같은 경우 handoff는 **적합한 도구가 아닙니다**:

- **병렬 작업 / worktree 분기.** 현재 세션이 살아있는 상태로 다른 곳에서
  작업하는 거라면 그건 handoff가 아니라 **작업 위임(delegation)** 입니다.
  스킬은 초안 작성 전에 이 점을 사용자에게 한 번 확인합니다.
- **프로젝트 문서화.** 그건 `docs/`에 들어갈 자료이지, 임시 디렉토리에
  저장될 상태 전달 산출물이 아닙니다.
- **세션 요약.** 그건 그냥 채팅에서 요약을 요청하시면 됩니다 — 스킬이
  필요 없습니다.

스킬은 이런 부적합 케이스를 감지하면 사용자에게 다시 묻도록 설계되어
있습니다.

```
claude-skill-handoff/
├── README.md                    ← 영문판
├── README.ko.md                 ← 지금 이 파일
├── LICENSE                      ← MIT
├── CHANGELOG.md
├── .gitignore
└── handoff/                     ← 스킬 본체; 이 폴더를 통째로 복사하면 설치 완료
    ├── SKILL.md
    └── assets/
        ├── draft_template.md    ← Step 3에서 사용
        └── final_template.md    ← Step 5에서 사용
```

`handoff/` 안의 모든 것이 스킬입니다. 그 바깥은 사람이 보는 레포 메타데이터
(README, LICENSE 등)입니다.

## 설계 원칙

포크하실 분들을 위해 알아두면 좋은 몇 가지 선택입니다:

- **사람의 개입(human-in-the-loop)은 타협 대상이 아닙니다.** Step 4(검증)는
  생략할 수 없습니다. 새 세션에 정말로 의미 있는 내용이 무엇인지 판단할
  수 있는 건 결국 사용자뿐이기 때문입니다.
- **저장 위치는 OS 임시 디렉토리 — 저장소가 아닙니다.** Handoff는 본질적으로
  휘발성입니다. Handoff 파일이 버전 관리가 필요할 만큼 오래 살아남았다면,
  그건 이미 문서화의 영역이지 handoff의 영역이 아닙니다.
- **길이 제한이 강제됩니다.** 비대한 handoff는 자기 목적을 무너뜨립니다 —
  새 세션이 매 턴 그 토큰 비용을 지불해야 하니까요. 목표는 150–400줄입니다.
- **시크릿은 기본적으로 마스킹.** API 키, 토큰, 비밀번호, PII는 절대 포함하지
  않습니다. 불확실하면 일단 마스킹하고 사용자에게 알립니다.

자세한 사양은 [`handoff/SKILL.md`](./handoff/SKILL.md)를 참고하세요.

## 기여하기

이슈와 PR을 환영합니다. 몇 가지 약속:

- `SKILL.md`는 500줄 이하로 유지해주세요 (Anthropic의 skill-creator 가이드라인).
- 템플릿을 추가하는 경우 `handoff/assets/`에 두고, `SKILL.md`에서 언제
  사용하는지 명시적으로 참조해주세요.
- 실제 문제를 해결하지 않는 한 `scripts/` 폴더는 추가하지 말아주세요.
  파일 저장은 `tempfile.gettempdir()` 한 줄로 충분하며, 추상화가 가치 있는
  영역이 아닙니다.
- PR 열기 전에 실제 세션에서 스킬을 발동해보세요. 이 도구는 체감이 중요합니다.
